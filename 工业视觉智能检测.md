# 工业视觉智能检测平台

### 项目简介

ValkyrieHub是一套集AI检测、毫秒级异步处理与数字化监控于一体的分布式工业视觉中枢

通过C++原生推理架构实现了对六大类钢材表面缺陷的智能拦截，并摆脱了对Python环境的依赖

系统依托自研的ByteBuffer网络通道、自实现的RPC通信协议与高性能异步线程池Kraken，在保障产线实时检测任务的同时，构建了全景式的数字化控制大屏，实现从底层的工业视觉感知到上层的生产数据决策的全链路闭环

技术选型，Qt6，QSS，C++11，MySQL5.7，MinIO，YOLOv8，Docker，WSL（Ubuntu22.4）

* **通信协议**

  为了保障Valkyrie系统在复杂网络环境下的大规模数据吞吐与实时性，使用双层分布式通信架构实现

  1. **边缘端与服务端：高性能二进制RPC**

     * **协议特征**：自研ByteBuffer封装
     * **设计原因**：为了追求极致的图片传输效率，选择基于原始字节流传输，直接操作内存缓冲区，避免通用协议中冗余的Base64编码开销，实现了视觉数据在分布式节点间的零损耗传输

  2. **服务端与可视化展示端：异步信令Rpc**

     * **协议特征**：4字节Header+JSON载荷
     * **设计原因**：在交互与数据展示环境，利用4字节长度消息头机制解决了TCP粘包和半包问题，并配合JSON格式的灵活性，实现了对多路产线节点状态的实时调度与异步刷新
  
* **核心模块**

  * **HermesRPC**

    它是一套基于C++11和TCP打造的高性能远程过程调用框架，主要负责前端检测节点与后端管理中心之间的双向指令调度与视觉数据回传

    **自实现高并发调度的Reactor模型**，底层基于Linux原生的Epoll多路复用技术实现水平触发事件循环，将所有网络I/O设置为非阻塞模式，彻底消除了传统同步模型在等待网络数据时的CPU资源消耗，**采用单线程单循环的并发架构**，主线程只负责连接的接入，随后将Socket句柄分发给子线程的EventLoop，这种架构支撑多台工业相机节点的高频并发接入

    **专为高吞吐打造的零拷贝ByteBuffer**，重写缓冲机制，解决传输高清图片引发的内存分配和拷贝导致的卡顿问题，**采用头部预留与零拷贝封装**，预留了16字节的消息头的存储空间，需要发送消息时，底层可以直接将协议头写入预留区，无需移动或拷贝庞大的图像数据，**双指针动态管理**，内置读写双指针，配合应用层的Codec架构，解决TCP字节流传输过程中的粘包和半包问题

    **RAII资源安全管理**，整个项目大量使用C++11智能指针与RAII范式，严格管理各资源的生命周期

    **提供多种协议解析**，自定义RPC、Modbus、Http...

  * **KakenPool**

    由于前端相机高频采集与后端YOLO深度学习推理存在巨大的速度差，直接处理极易导致主线程阻塞，所以摒弃了POSIX线程库以及存在隐患的无界队列，自实现了基于C++11的高性能并发引擎

    **使用C++11完美转发以及Move语义**，`cv::Mat`图像数据非常庞大，拷贝代价极其昂贵，所以采用了可变参数模板与完美转发，任务投递进入队列时，全链路支持Move语义，从根本上杜绝了任务流转过程中的冗余内存拷贝

    **策略模式拒绝机制，应对多种应用场景**，工业现场环境极端恶劣，如果摄像头每秒涌入大量数据，传统的无界队列会瞬间吃光内存导致OOM，所以Kraken引入**有界权限队列**，严格的边界控制，从源头切断内存溢出风险，并且通过权限队列实现了延时执行的功能，**动态拒绝策略**，借鉴企业级中间件结构，我实现了4种应对不同应用场景的拒绝策略拦截器

  * **SkuldVision**

    系统摒弃了传统机器视觉在复杂场景下的脆弱性，通过深度学习架构与C++底层推理的结合，实现了对金属表面缺陷的像素级拦截

    **项目采用YOLO架构**，引入深度学习前沿的目标检测架构，模型经过专业工业数据集训练，**四大缺陷锁定**，系统能够精确的锁定四大类复杂表面缺陷，并输出精确的物理边界框

    **算法与业务解耦**，将算法平台与业务逻辑解耦，能够随时装配更精确的大模型加以辅助

  * **ArgusLog**

    在极高并发的工业现场，使用传统的同步日志会严重拖垮主业务线程的吞吐量，因此引入自实现工业级异步日志系统，它以极低的性能开销，为底层Reactor网络库与AI模型线程提供了可靠的时序运行记录

    **双端解耦的生产者消费者模型**，彻底剥离日志的生成与持久化，前端的业务线程仅负责组装LogEvent并推入内存队列，随后返回消除了缓慢的磁盘I/O对主业务的拖拽，真正耗时的操作，由单例模式统一调度的后台专属线程异步批量执行
  
    **灵活的插件化分发网络**，抽象了底层的ILogHandler接口，支持轻松挂载多个输出端
    
    **提供宏定义，快速使用**
    
    ```c++
    #define LOG_DEBUG ::mnsx::argus::Logger(::mnsx::argus::LogLevel::DEBUG, __FILE__, __LINE__).stream()
    #define LOG_INFO  ::mnsx::argus::Logger(::mnsx::argus::LogLevel::INFO,  __FILE__, __LINE__).stream()
    #define LOG_ERROR ::mnsx::argus::Logger(::mnsx::argus::LogLevel::ERROR, __FILE__, __LINE__).stream()
    #define LOG_WARN  ::mnsx::argus::Logger(::mnsx::argus::LogLevel::WARN,  __FILE__, __LINE__).stream(
    ```
